# Project Documentation

This repository organizes design and planning documents into separate categories:

The main goal of `hipster-entity` is to provide strong metadata from interface-first entity definitions so other tools and higher-level systems can be built on top of that metadata.
Runtime helpers such as builders, proxies, factories, and other infrastructure components are layered building blocks, not mandatory parts of every adoption.

## Documentation sections

- [Brainstorm](brainstorm/README.md): exploratory design and generator proposal discussions.
- [Architecture](architecture/README.md): formal architecture decisions and stable modeling contracts.
- [Architecture decisions](architecture/DECISIONS.md): consolidated ADR index and decision records.
- [Roadmap](roadmap/README.md): implementation roadmap, status tracking, and policy updates.
- [User docs](user/README.md): end-user guides, usage walkthroughs, and troubleshooting help.
