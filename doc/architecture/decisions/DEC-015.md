# DEC-015: Generated field metadata method lookup strategy

- Status: Accepted
- Mission: establish a single, high-performance, generated method-name lookup algorithm for array-backed view proxies.
- Date: 2026-04-01
- Owners: project
- Related docs: [DEC-015 brainstorm](../../brainstorm/dec-015-generated-field-metadata-method-lookup-strategy.md), [DEC-010](DEC-010.md), [Entity interface design](../README.md)
- Supersedes: -

## Context

Generic proxy dispatch must map accessor method names to field ordinals on the hot path.
Initial implementations performed sorting in enum static initialization, which adds avoidable startup work.

Because field names are fully known during generation, lookup structures should be emitted precomputed.

## Decision

Use generated `switch(methodName)` mapping as the only supported field lookup strategy for method-name resolution.

The generator emits direct, source-visible switch code that returns the target field enum constant.
This avoids runtime sorting and avoids parallel lookup arrays in generated enums.

Runtime class initialization must not perform sorting for this mapping.

### Generated example shape

```java
public static PersonSummaryField forMethodName(String methodName) {
  if (methodName == null) {
    return null;
  }
  return switch (methodName) {
    case "id" -> id;
    case "firstName" -> firstName;
    case "lastName" -> lastName;
    case "age" -> age;
    case "departmentName" -> departmentName;
    case "metadata" -> metadata;
    default -> null;
  };
}
```

### Strategy scope

DEC-015 defines a single supported strategy:

- Generated `switch(methodName)` is the canonical and required method-name lookup path.

Other variants (first-char prefilter, sorted-array binary search, perfect hash) are experimental investigations and not part of this decision's required generator behavior.

## Consequences

- **Positive**:
  - Removes avoidable class-load sorting work.
  - Keeps lookup deterministic and source-visible in generated code.
  - Preserves generic proxy architecture without per-view handwritten switch code.
  - Allows incremental optimization path for larger schemas.
  - Can reduce generated data footprint by avoiding additional parallel lookup arrays.

## Field metadata iteration decision

We chose the most direct, low-overhead path for view metadata iteration:

- `ViewMeta` exposes enum-backed field accessors (via `fieldValues()`, `fieldNameAt()`, `fieldTypeAt()`).
- Serialization/deserialization loops iterate on `for (int i = 0; i < fieldCount(); i++)`.
- No separate `fieldNames` / `fieldTypes` arrays are stored in `ViewMeta` to avoid duplication.
- This maximizes JIT friendliness and minimizes GC in hot path.

Alternative approaches evaluated in brainstorm:

- using `forEach` callbacks over metadata (readability, slightly more indirection/lambda overhead)
- prebuilt `String[]` and `Class<?>[]` arrays in `ViewMeta` (copy/clone overhead)
- updating via `fieldType().getEnumConstants()` on every call (chosen as best compromise)

These alternatives are tracked in :
- `doc/brainstorm/dec-015-generated-field-metadata-method-lookup-strategy.md`

- **Negative**:
  - Large switch blocks can grow with field count and need generator maintenance when views are expanded.
  - Additional experimental strategies require separate evaluation and are not automatically enabled.

## Out of scope

- Selecting perfect-hash as the default for all views.
- Runtime global caches shared across entities.
- Mutation/change-tracking internals (covered by DEC-012 and DEC-014).

## Acceptance criteria

- Generated method mapping MUST default to source-visible string switch logic.
- Runtime enum initialization MUST NOT perform sort work for method mapping.
- Architecture docs MUST describe first-char prefilter as optional and benchmark-gated.
- Optional first-char bucketing MUST retain identical lookup semantics (same matches and misses) as default switch mapping.
- If optional strategies are enabled, generator MUST document chosen strategy and key input metrics (`n`, `u1`, `k1max`) in generated comments or metadata.
- Benchmark suite MUST include hit/miss distributions and clustered-prefix datasets before changing global default from switch mapping.
